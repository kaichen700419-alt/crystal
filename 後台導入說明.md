## ⚠️ 唯一需要工程師做一次的事：OAuth 登入中繼
GitHub Pages 上 Sveltia 登入需要一個 OAuth 中繼，二選一：

**做法 A（建議）Cloudflare Workers**
1. 部署官方 `sveltia-cms-auth`（GitHub 上有一鍵 Deploy）：https://github.com/sveltia/sveltia-cms-auth
2. GitHub → Settings → Developer settings → **OAuth Apps** → New，
   Authorization callback URL 填你的 worker 網址 + `/callback`
3. 把 Client ID / Secret 設進 worker 環境變數
4. 在 `admin/config.yml` 的 `backend:` 取消註解並填：
   `base_url: https://你的-worker.workers.dev`

**做法 B：改用 Cloudflare Pages / Netlify 部署**，用其內建 OAuth，省掉自建 worker。

## 上傳
把 `deploy-flat/` 內所有檔案（含 `admin/`、`images/`、`content.json`）上傳到 repo 即可。
> 註：`admin/` 與 `images/` 是資料夾，用 GitHub 網頁拖檔可能需要分批；建議用 GitHub Desktop 或 git 指令一次上傳整個資料夾結構。
